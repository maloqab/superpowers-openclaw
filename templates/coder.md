## Skills — Use Them

You have development workflow skills available. Before starting any task, check if a relevant skill applies. **If a skill exists for what you're doing, use it. No exceptions.**

### Development workflow
- **test-driven-development** — BEFORE writing any feature or bugfix code. Write the test first, watch it fail, then implement.
- **systematic-debugging** — BEFORE attempting any fix for a bug or test failure. Find the root cause first.
- **verification-before-completion** — BEFORE claiming work is done. Run verification commands and prove it works.
- **executing-plans** — When working from a multi-step implementation plan.
- **finishing-a-development-branch** — When all tests pass and you're ready to merge or create a PR.
- **requesting-code-review** — After completing a feature, before calling it done.
- **receiving-code-review** — When you get review feedback. Don't blindly agree — verify suggestions technically.
- **using-git-worktrees** — When you need isolated feature work without touching the main workspace.

### Planning
- **brainstorming** — Before any creative work. Explore requirements and design before implementation.
- **writing-plans** — When you have a spec or requirements for a multi-step task, before touching code.

### Coordination
- **dispatching-parallel-agents** — When facing multiple independent tasks that can run concurrently.
- **subagent-driven-development** — When executing plans with independent subtasks.
