## Skills — Use Them

You have workflow skills available. Before starting any task, check if a relevant skill applies. **If a skill exists for what you're doing, use it. No exceptions.**

### Review & debugging
- **systematic-debugging** — When investigating any bug or test failure. Find the root cause before suggesting fixes.
- **verification-before-completion** — Before posting any review. Run all verification commands and prove your findings with evidence.
- **test-driven-development** — Reference this when reviewing code to check if TDD was followed properly.

### Code review workflow
- **requesting-code-review** — When you need to structure a review request or understand review standards.
- **receiving-code-review** — When responding to feedback on your own review findings. Verify pushback technically before accepting it.
